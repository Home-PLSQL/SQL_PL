Слайд: Основные сведения

PL/SQL — процедурный блочно-структурированный язык.

  • Создание пакетов, процедур и функций, хранимых в базе данных.
  • Предоставление интерфейса для вызова внешних процедур.
  • Поддержка типов данных SQL и внутренних типов PL/SQL.
  • Работа с курсором.
  • Механизм обработки исключений.

Слайд: Основные сведения

  • Не чувствителен к регистру (кроме строковых переменных и констант).
  • Каждая конструкция PL/SQL должна заканчиваться символом ;
  • Одна конструкция может быть расположена на нескольких строках.

Слайд: Блоки PL/SQL

Блок — основная структурная единица PL/SQL.

Блок позволяет объединять объявления и операторы, связанные общей логикой.

  • Может быть анонимным или именованным.
  • Может содержать вложенные блоки.

Слайд: Блоки PL/SQL

Составные части блока:

  • секция объявлений;
  • тело блока (обязательная часть);
  • обработчики исключений.
DECLARE   
  -- Объявление переменных, типов, курсоров и проч. 
BEGIN   
  -- Код программы (обязательная часть блока)
EXCEPTION   
  -- Обработка исключений
END;
/* Многострочные
комментарии… */ 
-- Однострочный комментарий

Слайд: Блоки PL/SQL — пример

/*
  Простейший блок, реализующий вывод
  сообщения Hello world!
*/
BEGIN
    dbms_output.put_line('Hello world!');
END;

Слайд: Объявление переменных

Переменные:

  • Могут иметь тип данных SQL или PL/SQL.
  • Объявляются в секциях объявлений блоков PL/SQL.
  • Видны внутри блока, в котором они объявлены.
  • Получают тип данных при объявлении.
  • Могут быть проинициализированы значением при объявлении.
имя_переменной1 тип_переменной1; -- Переменная заданного типа
имя_переменной2 тип_переменной2 := значение2; -- С присвоением значения
имя_константы CONSTANT тип_константы := значение; -- Константы

Слайд: Объявление переменных — пример

DECLARE
    l_message1 VARCHAR2(100) := 'Первое сообщение';
    l_message2 VARCHAR2(100) := 'Второе сообщение';
BEGIN
    dbms_output.put_line(l_message1);
    dbms_output.put_line(l_message2);
    DECLARE
        l_message1 VARCHAR2(100) := 'Сообщение вложенного блока';
    BEGIN
        dbms_output.put_line(l_message1);
        dbms_output.put_line(l_message2);
    END;
    dbms_output.put_line(l_message1);
    dbms_output.put_line(l_message2);
END;

Слайд: Управляющие конструкции языка

  • Операторы выбора:
    • IF
    • CASE
  • Операторы цикла:
    • LOOP
    • FOR LOOP
    • WHILE LOOP
  • Операторы перехода:
    • GOTO

Слайд: Операторы выбора (IF)

  • IF-THEN-ELSIF-THEN-ELSE-END IF
IF условие1 THEN
  -- Набор действий для условия1
ELSIF условие2 THEN
  -- Набор действий для условия2
ELSE  
  -- Набор действий для случая
  -- если не выполнены оба условия
END IF;

Слайд: Операторы выбора (CASE)

  • CASE-WHEN-THEN-ELSE-END CASE
Первый вариант
CASE
  WHEN условие1 THEN
    -- Набор действий для условия1
  WHEN условие2 THEN
    -- Набор действий для условия2
  ELSE  
    -- Набор действий для случая
    -- если не выполнены оба условия
END CASE;

Слайд: Операторы выбора (CASE)

  • CASE-WHEN-THEN-ELSE-END CASE
Второй вариант
CASE переменная
  WHEN значение1 THEN
    -- Набор действий для значения1
  WHEN значение2 THEN
    -- Набор действий для значения2
  ELSE  
    -- Набор действий для случая если переменная
    -- не нашлась среди перечисленных значений
END CASE;

Слайд: Операторы выбора — пример

DECLARE
    l_num1 NUMBER;
    l_num2 NUMBER;
BEGIN
    l_num1 := 3;
    IF l_num1 = 2 THEN
        dbms_output.put_line('l_num1 равен 2');
    ELSE
        dbms_output.put_line('l_num1 не равен 2');
    END IF;
    IF l_num2 IS NULL THEN -- Сравнение l_num2 = NULL не работает
        dbms_output.put_line('l_num2 содержит пустое значение');
    ELSE
        dbms_output.put_line('l_num2 не содержит пустое значение');
    END IF;
    CASE NVL(l_num2, l_num1) -- Оператор NVL
        WHEN 2 THEN
            dbms_output.put_line('2!');
        WHEN 3 THEN
            dbms_output.put_line('3!');
        ELSE
            dbms_output.put_line('Другое!');
    END CASE;
END;

Слайд: Операторы цикла (простой LOOP)

  • LOOP-END LOOP
LOOP
  -- Набор действий
  EXIT WHEN условие; -- Выход из цикла
END LOOP;

Слайд: Операторы цикла (WHILE LOOP)

  • WHILE-LOOP-END LOOP
WHILE условие LOOP
  -- Набор действий
END LOOP;

Слайд: Операторы цикла (FOR LOOP)

  • FOR-LOOP-END LOOP
FOR переменная IN граница1..граница2 LOOP
  -- Набор действий
END LOOP;
В обратном порядке
FOR переменная IN REVERSE граница1..граница2 LOOP
  -- Набор действий
END LOOP;

Слайд: Операторы цикла — пример

DECLARE
    l_num NUMBER := 0;
BEGIN
    LOOP
        dbms_output.put_line('l_num = '|| l_num);
        l_num := l_num + 1;
        EXIT WHEN l_num > 5;
    END LOOP;
    l_num := 0;
    WHILE l_num <= 5 LOOP
        dbms_output.put_line('l_num = '|| l_num);
        l_num := l_num + 1;
    END LOOP;
    l_num := 0;
    FOR l_num IN 0..5 LOOP
        dbms_output.put_line('l_num = '|| l_num);
    END LOOP;
    dbms_output.put_line('l_num = '|| l_num);
END;

Слайд: Операторы перехода (GOTO)

  • GOTO
-- Блок какой-либо обработки
BEGIN
  -- Действия...
  IF условие THEN
    GOTO continue; -- Прерывание обработки и переход на метку continue
  END IF;
  -- Действия...
END;
<<continue>>

Не может применяться для перехода во вложенный блок или цикл.

Слайд: Операторы перехода — пример

DECLARE
    l_num NUMBER := 0;
BEGIN
    LOOP
        l_num := l_num + 1;
        IF l_num = 3 THEN
            GOTO continue;
        END IF;
        dbms_output.put_line('l_num = '|| l_num);
        EXIT WHEN l_num = 5;
        <<continue>> NULL;
    END LOOP;
END;

Слайд: Типы данных

Простые (скалярные) типы (простые значения без внутренних составляющих):

  • NUMBER(точность, масштаб) — число с плавающей точкой;
  • CHAR2(размер) — строка постоянной длины;
  • VARCHAR2(макс_размер) — строка переменной длины;
  • DATE — дата (со временем);
  • BOOLEAN — логическое значение;
  • и проч.

Слайд: Типы данных

Составные типы (структуры из определенных компонент):

  • TABLE — коллекция;
  • VARRAY — массив;
  • RECORD — запись.

Ссылочные типы: ссылка на объект, например REF CURSOR.

Слайд: Типы данных

LOB типы: «локатор», определяющий расположение больших объектов данных (графика, файлы,...):

  • BFILE — внешний двоичный файл;
  • BLOB — внутренний двоичный объект;
  • CLOB — внутренний символьный объект.

Слайд: Приведение типов

  • Явное:
    • TO_NUMBER()
    • TO_CHAR()
    • TO_DATE()
    • ...
  • Неявное.

Слайд: Типы данных — пример

DECLARE
  l_num NUMBER;
  l_char VARCHAR2(10);
  l_date DATE;
BEGIN
  l_num := 50;
  l_char := l_num; -- Неявно
  dbms_output.put_line('l_char = '|| l_char);
  l_char := '15.03.2010';
  l_date := TO_DATE(l_char, 'DD.MM.YYYY'); -- Явно (по маске)
  dbms_output.put_line('l_date = '|| l_date);
  l_char := TO_CHAR(l_date, 'YYYY-MM-DD'); 
  dbms_output.put_line('l_char = '|| l_char);
END;

Слайд: Коллекции

  • Таблицы PL/SQL (TABLE)
DECLARE
  TYPE тип_коллекция IS
    TABLE OF тип_элемента
    INDEX BY BINARY_INTEGER; -- Объявление нового типа-коллекции
  переменная тип_коллекция; -- Объявление переменной этого типа
BEGIN
  --...
END;
  • Массивы (VARRAY)
DECLARE
  TYPE тип_массив IS
    VARRAY(макс_размер) OF тип_элемента; -- Объявление нового типа-массива
  переменная тип_массив; -- Объявление переменной этого типа
BEGIN
  --...
END;

Слайд: Коллекции

Методы работы с коллекциями:

  • EXISTS(индекс_элемента) — существует ли элемент по индексу?
  • COUNT — количество элементов в коллекции.
  • LIMIT — размер varray-массива.
  • DELETE(с_элемента, по_элемент) — удалить элементы: все; один; диапазон.
  • FIRST, LAST — индексы первого и последнего элемента.
  • PRIOR(индекс_элемента), NEXT(индекс_элемента) — индексы следующего и предыдущего элемента.

Слайд: Коллекции — пример

DECLARE
    TYPE TStringArray IS TABLE OF VARCHAR2(100) INDEX BY BINARY_INTEGER;
    la_elements TStringArray;
    l_index NUMBER;
BEGIN
    la_elements(1) := 'Первый элемент';
    la_elements(2) := 'Второй элемент';
    la_elements(3) := 'Третий элемент';
    FOR i IN 1..la_elements.COUNT LOOP
        dbms_output.put_line(la_elements(i));
    END LOOP;
    la_elements(10) := 'Десятый элемент';
    l_index := la_elements.FIRST;
    LOOP
        dbms_output.put_line(la_elements(l_index));
        l_index := la_elements.NEXT(l_index);
        EXIT WHEN l_index IS NULL;
    END LOOP;
END;

Слайд: Записи

  • Записи (RECORD)
DECLARE
  TYPE тип_запись IS
    RECORD (поле1 тип1, поле2 тип2,...); -- Объявление нового типа-записи
  переменная тип_запись; -- Объявление переменной этого типа
BEGIN
  --...
END;

Слайд: Записи — пример

DECLARE
    TYPE TEmployee IS RECORD (ename VARCHAR2(10), job VARCHAR2(10));
    lr_emp TEmployee;
BEGIN
    lr_emp.ename := 'SCOTT';
    lr_emp.job := 'ANALYST';
    dbms_output.put_line(lr_emp.ename ||' '|| lr_emp.job);
END;

Слайд: Атрибуты типизации

  •  %TYPE
    • Объявление переменной с типом другой переменной.
    • Объявление переменной с типом поля таблицы в БД.
  •  %ROWTYPE
    • Объявление переменной типа запись, по структуре соответствующей строке таблицы в БД.
имя_переменной1 тип_переменной1; 
имя_переменной2 имя_переменной1%TYPE; -- Объявление переменной с типом другой переменной
имя_переменной3 имя_таблицы1.имя_поля%TYPE; -- Объявление переменной с типом поля таблицы
имя_переменной4 имя_таблицы2%ROWTYPE; -- Объявление переменной, соответствующей строке таблицы

Слайд: Атрибуты типизации — пример

DECLARE
    l_name emp.ename%TYPE;
    l_job emp.job%TYPE;
    lr_emp emp%ROWTYPE;
BEGIN
    l_name := 'JONES';
    l_job := 'MANAGER';
    lr_emp.ename := l_name;
    lr_emp.job := l_job;
    dbms_output.put_line(lr_emp.ename ||' '|| lr_emp.job);
END;

Слайд: Извлечение и обработка данных

  • Извлечение одной записи
DECLARE
  переменная1 таблица.поле1%TYPE;
  переменная2 таблица.поле2%TYPE;
BEGIN
  SELECT поле1, поле2 INTO переменная1, переменная2
    FROM таблица
    WHERE условия_отбора;
  -- Дальнейшие действия с переменными  
END;

Слайд: Извлечение данных — пример

DECLARE
    l_job emp.job%TYPE;
    l_sal emp.sal%TYPE;
    lr_emp emp%ROWTYPE;
BEGIN
    SELECT job, sal INTO l_job, l_sal
        FROM emp
        WHERE ename = 'SCOTT';
    dbms_output.put_line(l_job ||' '|| l_sal);
    SELECT * INTO lr_emp
        FROM emp
        WHERE ename = 'KING';
    dbms_output.put_line(lr_emp.job ||' '|| lr_emp.sal);
END;

Слайд: Извлечение и обработка данных

  • Извлечение набора записей
DECLARE
  TYPE тип_коллекция IS
    TABLE OF таблица.поле%TYPE
    INDEX BY BINARY_INTEGER;
  переменная тип_коллекция;
BEGIN
  SELECT поле BULK COLLECT INTO переменная
    FROM таблица
    WHERE условия_отбора;
  -- Дальнейшие действия с коллекцией
END;

Слайд: Извлечение данных — пример

DECLARE
    TYPE TEmpArray IS TABLE OF emp%ROWTYPE INDEX BY BINARY_INTEGER;
    la_emp TEmpArray;
BEGIN
    SELECT * BULK COLLECT INTO la_emp
        FROM emp
        WHERE sal > 2000;
    FOR i IN 1..la_emp.COUNT LOOP
        dbms_output.put_line(la_emp(i).ename ||' '|| la_emp(i).job ||' '|| la_emp(i).sal);
    END LOOP;
END;

Слайд: Курсоры

Курсор — получаемый при выполнении запроса результирующий набор записей плюс привязанный к нему указатель текущей записи.

Курсоры:

  • Явные (объявляются разработчиком).
  • Неявные (не требуют объявления, управляются автоматически).

Состояние курсора — атрибуты:

  •  %ISOPEN
  •  %FOUND
  •  %NOTFOUND
  •  %ROWCOUNT

Слайд: Курсоры

  • Организация работы с явным курсором
DECLARE
  CURSOR курсор IS
    SELECT поле1, поле2
      FROM таблица
      WHERE условия_отбора; -- Объявление курсора
  переменная1 таблица.поле1%TYPE;
  переменная2 таблица.поле2%TYPE;
BEGIN
  OPEN курсор; -- Открытие курсора
  LOOP -- Цикл по найденным записям
    FETCH курсор INTO переменная1, переменная2; -- Получение данных текущей записи
    EXIT WHEN курсор%NOTFOUND; -- Записи в курсоре закончились - выход из цикла
    -- Дальнейшие действия с переменными
  END LOOP;
  CLOSE курсор; -- Закрыть курсор
END;

Слайд: Курсоры

  • Цикл FOR-LOOP
DECLARE
  CURSOR курсор IS
    SELECT поле1, поле2
      FROM таблица
      WHERE условия_отбора; -- Объявление курсора
BEGIN
  FOR запись IN курсор LOOP -- Цикл по найденным записям
    -- Действия с записью в виде:
    -- запись.поле1, запись.поле2
  END LOOP;
END;

Слайд: Вызовы SQL в PL/SQL-ном блоке

Варианты вызова:

  • Статический («встроенный» SQL): SQL-операторы формируются на этапе компиляции.
  • Динамический: SQL-операторы формируются во время выполнения (в виде текстовых переменных).
    • Если текст запроса заранее не может быть известен.
    • Если запрос содержит операторы DDL.

Слайд: Вызовы SQL в PL/SQL-ном блоке

  • Bind-переменные
  --...
  EXECUTE IMMEDIATE
    'SELECT поле1 FROM таблица WHERE поле2 = :1'
  INTO переменная1 USING переменная2;
  --...
  OPEN курсор FOR
    'SELECT поле1 FROM таблица WHERE поле2 = :1'
  USING переменная2;
  --...

Слайд: Динамический SQL — пример

DECLARE
    TYPE TCursor IS REF CURSOR;
    lc TCursor;
    l_select VARCHAR2(100) := 'SELECT * FROM emp';
    l_where VARCHAR2(100);
    lr_emp emp%ROWTYPE;
BEGIN
    l_where := 'WHERE ename = :1';
    EXECUTE IMMEDIATE l_select ||' '|| l_where
        INTO lr_emp USING 'MARTIN';
    dbms_output.put_line(lr_emp.ename);
    l_where := 'WHERE deptno = :1';
    OPEN lc FOR l_select ||' '|| l_where
        USING 30;
    LOOP
        FETCH lc INTO lr_emp;
        EXIT WHEN lc%NOTFOUND;
        dbms_output.put_line(lr_emp.ename);
    END LOOP;
    CLOSE lc;
END;

Слайд: Управление изменениями

Транзакция — последовательность действий с БД, которая:

  • или полностью фиксируется в БД,
  • или полностью отменяется.

Все SQL-операторы входят в одну транзакцию до тех пор, пока не будет сделана ее фиксация (COMMIT) или откат (ROLLBACK).

Слайд: Обработка ошибок

Исключение — ошибка времени выполнения.

  • Нормальное выполнение блока прекращается.
  • Управление возвращается внешнему (вызвавшему) блоку.
  • ...до тех пор, пока исключение не будет «перехвачено» и обработано:
    • выдано user-friendly сообщение пользователю;
    • записано в лог;
    • проигнорировано;
    • ...
BEGIN   
  -- Код программы, в котором ожидается возникновение ошибки
  -- ...
EXCEPTION WHEN исключение THEN -- Идентификатор перехватываемого исключения
  -- Обработка исключения
END;

DECLARE
    l_dept VARCHAR2(100);
    PROCEDURE get_dept(a_ename VARCHAR2, a_dept OUT VARCHAR, a_needed BOOLEAN := FALSE)
    AS
    BEGIN
        SELECT dname INTO a_dept
            FROM dept
            WHERE deptno = (SELECT deptno FROM emp WHERE ename = a_ename);
    EXCEPTION WHEN NO_DATA_FOUND THEN
        IF a_needed THEN
            RAISE;
        END IF;

    dbms_output.put_line(l_dept);
    get_dept('SKOTT', a_needed => TRUE, a_dept => l_dept);
    dbms_output.put_line(l_dept);
END;

/* Пакет реализует API доступа к данным сотрудников:
- получение имени сотрудника по его номеру

Слайд: Пакеты PL/SQL — пример

CREATE OR REPLACE PACKAGE BODY pk_emp AS
    TYPE TStringArray IS TABLE OF VARCHAR2(100) INDEX BY BINARY_INTEGER;
    la_emp TStringArray;
    la_dept TStringArray;
    FUNCTION get_name(a_empno NUMBER) RETURN VARCHAR2
    AS
    BEGIN
        IF NOT la_emp.EXISTS(a_empno) THEN
            SELECT ename INTO la_emp(a_empno)
                FROM emp
                WHERE empno = a_empno;
        END IF;
        RETURN la_emp(a_empno);
    END get_name;
    FUNCTION get_dept(a_empno NUMBER) RETURN VARCHAR2
    AS
    BEGIN
        RETURN la_dept(a_empno);
    END get_dept;
BEGIN
    FOR lc IN (SELECT empno, dname FROM emp e, dept d WHERE d.deptno = e.deptno)
    LOOP
        la_dept(lc.empno) := lc.dname;
    END LOOP;
END pk_emp;

Утилита SQL*Plus: позволяет выполнять команды SQL и блоки PL/SQL.

  • sqlplus — работа в окне командной строки
  • sqlplusw — оконная версия

Запуск: 

sqlplus [опции] [регистрационная информация] [начальный сценарий]

Опции:

  • -H(ELP)
  • -V(ERSION)
  • -S(ILENT)
  • ...

Слайд: Коротко об sqlplus

Регистрационная информация:

  • "пользователь/пароль@база_данных" AS роль
  • /NOLOG — запуск без подключения (обычно для выполнения сценариев)

Начальный сценарий: файл с командами SQL*Plus, операторами SQL и анонимными PL/SQL-блоками, выполняемый при запуске.

Запуск sqlplus — пример: 

sqlplus /NOLOG @script.sql param1 param2
  • param1, param2: параметры сценария

Слайд: Коротко об sqlplus — пример

Пример файла сценария: 

/*script.sql*/
-- Определить код возврата при ошибке
WHENEVER SQLERROR EXIT -1 ROLLBACK
-- Подключиться к БД (параметризовано)
CONNECT &1/&2@ORAXE;
-- Включить вывод
SET SERVEROUTPUT ON;
-- Логировать действия в файл
SPOOL test.log;
-- Объявить и присвоить bind-переменную     
var v_cnt NUMBER;
EXEC :v_cnt := 30;
-- Выполнить команду SQL
SELECT * FROM emp WHERE deptno = :v_cnt;
-- Выполнить блок PL/SQL
BEGIN
    dbms_output.put_line('Listed employees for deptno '|| :v_cnt);
END;
/
-- Выйти с кодом "успешно"
EXIT 0;
