Вводная

В ходе лекций в качестве примера PL/SQL пакетов был показан пакет pk_emp. 

/* Пакет реализует API доступа к данным сотрудников:
- получение имени сотрудника по его номеру
- получение названия подразделения сотрудника по его номеру
Обращения к БД кешируются.
*/
CREATE OR REPLACE PACKAGE pk_emp AS
    FUNCTION get_name(a_empno NUMBER) RETURN VARCHAR2;
    FUNCTION get_dept(a_empno NUMBER) RETURN VARCHAR2;
END pk_emp;
/
CREATE OR REPLACE PACKAGE BODY pk_emp AS
    TYPE TStringArray IS TABLE OF VARCHAR2(100) INDEX BY BINARY_INTEGER;
    la_emp TStringArray;
    la_dept TStringArray;
    FUNCTION get_name(a_empno NUMBER) RETURN VARCHAR2
    AS
    BEGIN
        IF NOT la_emp.EXISTS(a_empno) THEN
            SELECT ename INTO la_emp(a_empno)
                FROM emp
                WHERE empno = a_empno;
        END IF;
        RETURN la_emp(a_empno);
    END get_name;
    FUNCTION get_dept(a_empno NUMBER) RETURN VARCHAR2
    AS
    BEGIN
        RETURN la_dept(a_empno);
    END get_dept;
BEGIN
    FOR lc IN (SELECT empno, dname FROM emp e, dept d WHERE d.deptno = e.deptno)
    LOOP
        la_dept(lc.empno) := lc.dname;
    END LOOP;
END pk_emp;
/

Цели:

  • Расширить функциональность пакета в соответствии с задачами.
  • Протестировать его работоспособность.

Задачи

Задача 1

User friendly обработка ошибок.

Модифицировать функции pk_emp.get_name() и pk_emp.get_dept() таким образом, чтобы для несуществующего empno возвращалось пустое значение (NULL), а не возникала ошибка Oracle.

Аналогичный подход реализовывать в дальнейших задачах.

Задача 2

Поиск данных сотрудника по имени.

Реализовать функцию, которая для заданного имени сотрудника возвращает строку, состоящую из его имени, должности и названия отдела. Для получения названия отдела использовать уже имеющийся функционал.

Задача 3

Построение отчета о сотрудниках.

Реализовать процедуру, которая принимает на вход:

  • список полей таблицы emp, которые нужно вывести в отчет;
  • список полей таблицы emp, по которым отчет должен быть отсортирован;
  • разделитель столбцов в отчете.

Списки полей задаются в строку через запятую. Если разделитель не задан, то столбцы разделяются символом «|».

Использовать динамический SQL и цикл по курсору. Для вывода отчета использовать dbms_output.put_line().

Задача 4

Проверочные функции.

Реализовать функции, которые бы проверяли:

  1. Работают ли в заданном городе сотрудники с заданной должностью?
  2. Есть ли на заданной должности сотрудники, получающие оклад заданного грейда?
  3. Является ли первый заданный сотрудник подчиненным второго заданного сотрудника (прямым либо косвенным)?

На выходе должно быть TRUE или FALSE.

Задача 5

Обработка повышения зарплаты сотрудников.

Реализовать процедуру, которая для заданных сотрудников:

  1. Проверяет возможность повышения зарплаты на заданный процент.
  2. Выводит имя сотрудника и — если повышение разрешено — новую сумму, если нет — текст «не повышать».

Критерием проверки взять то, что после повышения зарплата не должна оказаться больше зарплаты непосредственного руководителя сотрудника.

Предусмотреть возможность задать на вход или определенного сотрудника, или целиком отдел. Для этого на выбор: создать перегрузки процедуры с разными наборами параметров; либо обойтись одной процедурой, но имеющей необязательные параметры для номера сотрудника и номера отдела.

Задача 6

Отладка и тестирование.

Написать PL/SQL блок, в котором были бы вызовы реализованных функций и процедур пакета pk_emp с различными характерными значениями параметров — с тем чтобы можно было проверить правильность их работы. Для вывода результатов использовать dbms_output.put_line().

Например для первой проверочной функции из задачи 4 (job_for_town) можно вывести: 

pk_emp.job_for_town('NEW YORK', 'ANALYST') вернул FALSE
pk_emp.job_for_town('CHICAGO', 'SALESMAN') вернул TRUE
